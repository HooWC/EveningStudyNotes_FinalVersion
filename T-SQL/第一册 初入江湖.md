```sql
-- 1. 建数据库
CREATE DATABASE MyAppDB;
GO

-- 切换到数据库
USE MyAppDB;
GO

-- 2. 建 Users 表（常见字段）
CREATE TABLE dbo.Users (
    UserID INT IDENTITY(1,1) PRIMARY KEY,
    FullName NVARCHAR(100) NOT NULL,
    Img NVARCHAR(255) NULL,
    NRIC NVARCHAR(50) NULL,
    Age INT NULL,
    BirthDate DATE NULL,
    Gender NVARCHAR(10) NULL,
    Email NVARCHAR(255) NOT NULL UNIQUE,
    Active BIT NOT NULL DEFAULT (1),
    CreatedAt DATETIME2 NOT NULL DEFAULT SYSUTCDATETIME()
);
GO

-- 3. 插入一条测试数据
INSERT INTO dbo.Users (FullName, Img, NRIC, Age, BirthDate, Gender, Email, Active)
VALUES ('測試用戶', NULL, '900101-01-0001', 35, '1990-01-01', 'Male', 'test.user@example.com', 1);
GO

-- 4. 查看数据
SELECT * FROM dbo.Users;
GO

-- & 更新
UPDATE Users
SET FullName = N'测试用户'
WHERE UserID = 1;
```

```
1. IDENTITY(1,1)

用在 UserID INT IDENTITY(1,1) PRIMARY KEY。
表示这是一个 自增列。
第一个数字 1 = 起始值（从 1 开始）。
第二个数字 1 = 每次增加的步长（每插入一行，值 +1）。
👉 例子：插入 3 行数据后，UserID 会分别是 1, 2, 3。
```

```
2. BIT NOT NULL DEFAULT (1)

BIT 是布尔类型，值只能是 0 或 1（SQL Server 内部其实允许 0,1, NULL）。
NOT NULL 表示不能为 NULL。
DEFAULT (1) 表示默认值是 1。
👉 例子：Active BIT NOT NULL DEFAULT (1)
不写 Active 的时候，系统会自动填 1（表示活跃/启用）。
你也可以插入 0 表示不活跃。
```

```
3. DATETIME2

是 SQL Server 提供的更精确的日期时间类型。
支持 公元 0001-01-01 到 9999-12-31，精度可达 100 纳秒。
比 DATETIME 范围更大、精度更高。
👉 适合存储时间戳，例如创建时间、更新时间。
```

```
4. DEFAULT SYSUTCDATETIME()

DEFAULT = 设置默认值。
SYSUTCDATETIME() = 返回当前系统的 UTC 时间（世界标准时间，不受时区影响）。
比 GETDATE() 精确，推荐用于时间戳。
👉 例子：当插入一条记录时，如果没指定 CreatedAt，它会自动填上插入时的 UTC 时间。
```

```
🔑 总结：

IDENTITY(1,1) → 自增主键（从 1 开始，每次 +1）。
BIT NOT NULL DEFAULT (1) → 布尔字段，默认启用状态。
DATETIME2 → 高精度日期时间类型。
DEFAULT SYSUTCDATETIME() → 默认存储当前 UTC 时间。
```

```
SQL 语句里忘了加 N 前缀
在 SQL Server 里，如果要插入 Unicode（中文、日文、韩文等），必须写 N'...'。
否则 SQL Server 会把它当成普通字符串处理，结果中文丢失。
```
